# 프로토콜 (Protocol)

  • 통신 규칙
  • 통신 절차를 규정하고 해당 절차에 이름을 붙이는 것
  • 아주 유명한 프로토콜들은 이름이 있다(ftp, http, helnet, smtp, ...)

# HTTP 프로토콜의 작동 순서

  1. 사용자(클라이언트)가 웹 브라우저 주소창에 보고싶은 페이지의 URL을 입력함 링크를 클릭하는 것도 주소창에 URL을 입력하는 것과 같다
  2. 웹 브라우저는 URL에 포함되있는 서버의 IP주소를 활용하여 서버 컴퓨터에 요청(request)을 보낸다
  3. nslookup naver.com
  4. 요청을 받은 웹 서버는 클라이언트가 보낸 URL에 포함되어 있는 URI를 보고 사용자가 원하는 자원을 해당 클라이언트에게 응답(Response)해준다
  5. 요청을 보냈던 클라이언트의 웹 브라우저는 웹 서버의 응답을 받아서 사용자에게 해당 코드를 해석하여 보여준다

# DNS (Domain Name Service)

  • URL의 IP주소 자리에 IP주소 대신 사용할 수 있는 이름
  • 사용자가 요청을 보낼 때 도메인 이름을 사용한다면 DNS 서버에서 해당 도메인을 IP주소로 치환한 후에 해당 웹 서버를 찾아감
  • DNS 서버는 각 국가의 ISP(Internet Service Provider)가 관리함

# IP 주소

  • 각 컴퓨터를 찾아가기 위한 인터넷 상의 주소
  • 모든 컴퓨터에 IP주소를 할당하면 부족할 수도 있으므로 공인IP(외부와 통신할 수 있는 IP)와 사설IP(내부망에서만 사용되는 IP)라는 개념이 있음
  • IPv4는 0.0.0.0 ~ 255.255.255.255까지 사용가능
  • IPv4의 모든 IP주소가 고갈될 때를 대비해 IPv6가 준비되어 있음

# 포트 번호

  • 각 컴퓨터의 어떤 프로그램과 통신하는지 식별하기 위한 번호
  • 포트번호는 0 ~ 65535번까지 사용할 수 있음 2byte
  • 고정적으로 사용하는 잘 알려진 포트번호들이 존재함
  • 80: http, 443: https, 21: ftp, 23: telnet ...
